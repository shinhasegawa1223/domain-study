# サブドメイン

ドメイン（業務領域）を細分化したもの。大きなシステムを理解しやすい単位に分割する。

---


## 3種類のサブドメイン

| 種類 | 説明 | 開発戦略 |
|------|------|----------|
| **コアサブドメイン** | ビジネスの競争優位性を生む中核 | 自社で注力して開発 |
| **サポートサブドメイン** | コアを支援するが差別化要因ではない | 自社開発だが優先度は低め |
| **汎用サブドメイン** | どの企業でも同じような機能 | 外部サービスで代用 |

| 種類 | 競争優位性 | 投資優先 |リソース|
|------|------|----------|---|
| **コアサブドメイン** | ビジネスの競争優位性を生む中核 | 最優先 | 優秀な人材と時間、予算 |
| **サポートサブドメイン** | コアを支援するが差別化要因ではない | 低い | そこまで |
| **汎用サブドメイン** | どの企業でも同じような機能 | 中 | 一定のスキル適度な予算 |



## 具体例（ECサイトの場合）

| サブドメイン | 種類 | 理由 |
|-------------|------|------|
| 商品レコメンド | コア | 独自アルゴリズムで差別化 |
| 価格戦略 | コア | 競合との差別化ポイント |
| 在庫管理 | サポート | 必要だが差別化要因ではない |
| 顧客管理 | サポート | 必要だが差別化要因ではない |
| 決済 | 汎用 | Stripe等で代用可能 |
| 認証 | 汎用 | Auth0等で代用可能 |
| メール送信 | 汎用 | SendGrid等で代用可能 |

## なぜ分類するのか？

| 観点 | 効果 |
|------|------|
| **リソース配分** | コアサブドメインに優秀なエンジニアを集中 |
| **コスト削減** | 汎用サブドメインは外部サービスを活用 |
| **優先順位** | どこに投資すべきか明確になる |

> [!IMPORTANT]
> コアサブドメインは企業のビジネス価値の源泉。ここに最も注力すべき！


## 注意
環境や市場の変換によって戦略やルールなどが変わるため定期に的に確認、見直しを行うことが重要である。
